React创建组件方式

发表于 2018-12-28 | 分类于 react | | 阅读次数:

react推出后，先后出现了三种定义react组件的方式，殊途同归，具体的三种方式：

  1. 函数式定义 无状态组件
  2. es5原生方式 React.creatClass组件
  3. es6形式 extends React.Component定义组件

既然有三种方式可以定义，那为什么会有这么多种定义方式，所以这三种定义方式有什么不同的地方？

无状态函数式组件

它是为了创建纯展示组件，这种组件只负责根据传入的props来展示，不涉及到state状态操作。

这种组件代码量少，轻量，简单，更容易理解。当你看到一个无状态组件的时候，应该联想到它就是一个纯函数，它只是接收属性props，渲染界面，不执行一些与UI无关的逻辑处理，同样的props会渲染出相同的结果，不会有副作用。

因为这种组件不会被实例化，所以不需要分配多余的内存，能让性能有一定的提升。不过因为没有实例化过程，所以无法访问组件中的对象，例如 this.state,this.setState，this.ref ，所以无法访问生命周期的方法和状态管理。

React.Component

它是用ES6 class来创建有状态组件，已经取代React.createClass形式，相对于React.createClass，可以更好的被复用。

React.createClass与React.Component区别

this绑定

React.createClass创建的组件，其每一个成员函数的this都有React自动绑定，任何时候使用，直接使用this.method即可，函数中的this会被正确设置.

React.Component创建的组件，其成员函数不会自动绑定this，需要开发者手动绑定，否则this不能获取当前组件实例对象。

当然，React.Component有三种手动绑定方法：可以在构造函数中完成绑定，也可以在调用时使用method.bind(this)来完成绑定，还可以使用arrow function来绑定。拿上例的handleClick函数来说，其绑定可以有：

组件初始状态state

React.createClass创建的组件，其状态state是通过getInitialState方法来配置组件相关的状态；
React.Component创建的组件，其状态state是在constructor中像初始化组件属性一样声明的。

Mixins

React.createClass 在创建组件的时候使用mixins属性到达复用

而React.Component则不支持这个属性，但是提供了高阶组件来代替它
